Overview

Production-Ready Shipping Integration

The Lift Commerce Connector is a FastAPI application that automates order fulfillment between Odoo 17 and Lift Commerce's shipping API. It provides real-time order monitoring, automated processing, and comprehensive tracking synchronization.

Key Features

  • Automated Order Processing - Monitors Odoo for new orders every 5 minutes
  • Secure Access - HTTP Basic Authentication protects all endpoints
  • Real-time Metrics - Dashboard shows processing statistics and status
  • Tracking Integration - Webhook receives tracking updates from Lift Commerce
  • Error Handling - Comprehensive retry logic and failure management

Architecture

System Components
  • FastAPI Application - Async web server
  • Odoo Integration - XML-RPC connection
  • Lift Commerce API - RESTful integration
  • Background Tasks - Automated monitoring
  • Web Dashboard - Real-time interface
Data Flow
  1. Order Detection from Odoo
  2. Data Transformation & Validation
  3. Shipping Request to Lift Commerce
  4. Tracking Updates via Webhook
  5. Status Synchronization

Configuration

Security Note: Always use environment variables for sensitive configuration data in production.

Dashboard Configuration

Configure Lift Commerce settings through the web interface:

API URL: https://api.liftcommerce.com
API Key: Your Lift Commerce authentication key
Warehouse ID: Target warehouse for fulfillment

Odoo Connection

# Environment Variables
ODOO_URL=https://your-instance.odoo.com
ODOO_DB=your-database-name
ODOO_USERNAME=your-api-user
ODOO_API_KEY=your-api-key

Authentication

Login:
Username: configured via ADMIN_USERNAME
Password: configured via ADMIN_PASSWORD_HASH (SHA-256)

API Endpoints

Public Endpoints
  • GET /health - System health check
  • POST /webhook/tracking - Tracking updates
Protected Endpoints
  • GET / - Main dashboard
  • GET /documentation - This documentation
  • GET /orders/ready - Fetch orders
  • POST /orders/{id}/sync - Sync order
  • POST /config - Update config
  • GET /sync/status - Sync history

API Usage Examples

# Health Check
curl http://your-server/health

# Fetch Orders (with auth)
curl -u admin:YOUR_PASSWORD http://your-server/orders/ready

# Manual Sync
curl -u admin:YOUR_PASSWORD -X POST http://your-server/orders/123/sync

Background Tasks

Order Monitoring Task

Frequency: Every 5 minutes

Function:

  • Detect new orders in 'sale' state
  • Filter unprocessed orders
  • Transform data with defaults
  • Submit to Lift Commerce API
  • Update order status in Odoo
Tracking Update Task

Frequency: Every 10 minutes

Function:

  • Check for tracking updates
  • Sync tracking numbers to Odoo
  • Update delivery status
  • Process webhook data

Troubleshooting

# Check service logs
sudo journalctl -u lift-commerce-connector -f

# Verify dependencies
source /opt/lift-commerce-connector/venv/bin/activate
pip list

# Check file permissions
ls -la /opt/lift-commerce-connector/

  • Verify Odoo URL and database name in configuration
  • Check API key permissions in Odoo user settings
  • Ensure firewall allows XML-RPC connections (port 443/80)
  • Test connection: curl https://your-odoo.com/xmlrpc/2/common

  1. Check order states in Odoo (must be 'sale' state)
  2. Verify Lift Commerce API credentials in dashboard
  3. Review background task logs: grep "order_monitoring" /var/log/lift-commerce-connector.log
  4. Test manual sync from dashboard
